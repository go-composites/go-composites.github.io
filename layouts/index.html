golang-cop

Small, hand-written object-oriented primitives for Go — a Result/Option value type, a null-object Error, and array and string types that thread failures through one shared Result channel.

pure Go null-object pattern Result / Option value type BSD-3-Clause early & experimental
Get started GitHub

golang-cop is an experiment in writing Go in an explicitly object-oriented style: every package exposes an Interface with a New constructor, and operations return a Result rather than Go's idiomatic multiple-return error. The null-object pattern runs throughout — a fresh Result, Error or Null is a real, non-nil value — so callers branch on HasError() instead of nil checks.

The packages build on each other: result sits on null and error; array's methods all return a Result; and string.Split returns a Result wrapping an array of strings. This is early, evolving code — no CI gate, coverage badge or SIMD is claimed here.
